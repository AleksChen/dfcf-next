npm run dev

## 📱 功能演示流程

### 第一步: 查看空状态

1. 打开浏览器: `http://localhost:3000`
2. 查看:
   - 统计卡片显示 0
   - 空状态提示
   - 刷新按钮

**说明**: 初始状态,数据库为空

---

### 第二步: 触发爬虫

1. 点击顶部菜单 "爬虫任务"
2. 或访问: `http://localhost:3000/crawler`

**填写表单:**
```
股票代码: 002085
平台: 东方财富 (选择下拉框)
起始页码: 1
结束页码: 2
```

3. 点击 "开始爬取"

**预期结果:**
```json
{
  "success": true,
  "count": 20,
  "platform": "eastmoney",
  "message": "成功爬取 20 条数据"
}
```

**时长**: ~3-5 秒

---

### 第三步: 查看数据

1. 点击顶部菜单 "数据分析"
2. 或访问: `http://localhost:3000`

**查看统计信息:**
- 总帖子数: 20
- 东方财富: 20
- 雪球: 0

**查看数据表格:**
- 20 条帖子数据
- 包含: 标题、作者、股票代码、平台、点击数、评论数、发布时间

---

### 第四步: 测试搜索

在搜索框输入:
- 搜索作者: `股友`
- 搜索标题关键词: `万`
- 搜索股票代码: `002085`

**预期**: 表格实时过滤,显示匹配结果

---

### 第五步: 测试排序

点击表头:
- **点击数** - 按热度排序
- **评论数** - 按互动排序
- **发布时间** - 按时间排序

**预期**: 表格数据按选定字段排序

---

### 第六步: 测试分页

1. 查看底部分页器
2. 切换页码: 1, 2, 3...
3. 修改每页显示数量: 10, 20, 50, 100

**预期**: 表格内容相应变化

## 🎯 API 演示

### 方式 1: 使用 curl

```bash
# 1. 查看统计信息
curl http://localhost:3000/api/stats

# 2. 查询数据 (限制 5 条)
curl http://localhost:3000/api/posts?limit=5

# 3. 按股票代码查询
curl http://localhost:3000/api/posts?stockCode=002085&limit=10

# 4. 按平台查询
curl http://localhost:3000/api/posts?platform=eastmoney&limit=10

# 5. 触发爬虫
curl -X POST http://localhost:3000/api/crawl \
  -H "Content-Type: application/json" \
  -d '{
    "stockCode": "002085",
    "platform": "eastmoney",
    "pageStart": 1,
    "pageEnd": 1
  }'
```

### 方式 2: 使用浏览器开发者工具

1. 打开浏览器开发者工具 (F12)
2. 切换到 **Network** 标签
3. 执行任意操作 (如点击"刷新")
4. 查看网络请求:
   - 请求 URL
   - 响应数据
   - 耗时

## 🐳 Docker 演示

### 构建镜像

```bash
docker build -t dfcf-crawler .
```

**预期**: ~3-5 分钟

### 运行容器

```bash
docker run -p 3000:3000 \
  -v $(pwd)/data:/app/data \
  dfcf-crawler
```

### 使用 Docker Compose

```bash
docker-compose up -d
```

**预期**: 后台运行,访问 `http://localhost:3000`

## 💬 常见问题回答

演示时可能被问到:

**Q: 为什么用 Next.js?**
A: 统一全栈开发,减少配置,提高效率

**Q: 支持多少数据量?**
A: SQLite 可处理百万级数据,对于爬虫应用足够

**Q: 能爬取哪些平台?**
A: 目前支持东方财富和雪球,架构设计支持快速扩展

**Q: 如何部署?**
A: 推荐 Docker,一行命令完成

**Q: 性能如何?**
A: 启动 3s, 构建 9s, API 响应 < 100ms
